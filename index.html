What is AI & ML?

The 30-second version. Artificial Intelligence is the broad idea of getting computers to do things that usually need human thinking — reading, talking, deciding, recognising. Machine Learning is the specific recipe that powers nearly all modern AI: instead of writing the rules by hand, you show a model lots of examples and let it figure out the rules itself.

🧠

Artificial Intelligence (AI)

the umbrella term

Any technique that lets machines mimic human-like behaviour — playing chess, recognising faces, translating languages, writing essays. Rule-based bots, search engines, and chatbots all live under this tent.

📊

Machine Learning (ML)

learn from examples

The subset of AI where the computer is shown data (emails labelled spam/not-spam, photos of cats and dogs) and statistically learns the pattern. No hand-written rules — the model discovers them.

🕸️

Deep Learning

stacked neural networks

ML done with multi-layer neural networks. When the layers get really deep and the data really big, you start to see GPT, Claude, Gemini, Llama — the Large Language Models (LLMs).

🗣️

Generative AI & LLMs

predict the next token

LLMs are deep neural nets trained on enormous text corpora. They generate by guessing the next word, one token at a time. Brilliant at language — but boxed in by their training cut-off and a fixed context window. That limitation is exactly what RAG, MCP, and Agentic RAG were built to break.

In one sentence
AI → ML → Deep Learning → LLMs → (RAG · MCP · Agentic RAG)

Each step is a subset of the one before. The decks in this hub focus on the final layer — making LLMs useful in the real world by giving them access to your data, your tools, and the ability to act.

The three problems LLMs alone can't solve

⏳Stale knowledge

  • An LLM only knows what was in its training data
  • It doesn't know about events after its cut-off date
  • It can't see your company's internal documents
  • It has no clue about today's stock price or weather

🎭Hallucinations

  • When unsure, LLMs confidently make things up
  • Fake citations, invented API names, wrong numbers
  • Sounds perfectly fluent — and is perfectly wrong
  • You can't tell guess from fact without a source

🧤Can't take actions

  • A vanilla LLM only outputs text
  • It can't query your database, send email, file a ticket
  • It can't open a file, run code, or call an API
  • It needs tools wired up to do anything in the world

🛠️That's where RAG · MCP · Agentic RAG come in

  • RAG fixes stale knowledge & hallucinations by retrieving your real documents
  • MCP standardises how LLMs talk to data sources and tools
  • Agentic RAG lets the LLM plan, decide, and act over many steps
  • Each builds on the previous — read in order for the smoothest ride

You can read the three decks in any order, but this is the path designed for someone with zero background. Each step builds the vocabulary you need for the next.

The Model Context Protocol is the open standard (Anthropic, Nov 2024) for connecting LLMs to data, tools, and prompts. Once you've seen why grounding matters in RAG, MCP shows you the clean way to deliver that grounding at scale — the "USB-C port for AI apps".

≈ 45–60 min 14 sections recommended after RAG

Finish with Agentic RAG — Zero to Hero

Combine retrieval (RAG) with autonomous reasoning, planning, tool-use, and memory. You'll meet the four agentic patterns, agentic workflows, the memory layer, and end-to-end ADW (Agentic Document Workflows). This is the frontier — read it last for maximum payoff.

≈ 60–75 min 8 sections advanced

How they fit together

A side-by-side cheat sheet for the moments when you need to remember which tool solves which problem.

Concept What it solves Core primitive When you use it Maturity
LLM (alone) Fluent text generation Token prediction Open-ended writing, casual Q&A stable
RAG Stale knowledge, hallucinations Vector retrieval + prompt grounding Search-then-answer over your docs production
MCP Bespoke per-integration glue JSON-RPC 2.0 standard (tools/resources/prompts) Plugging any LLM app into any data source / tool new standard (Nov 2024)
Agentic RAG Single-shot retrieval is not enough Plan → retrieve → reflect → act → loop Multi-step research, decision workflows, ADW frontier

Visual: where each layer plugs in

📚Knowledge layer — RAG

  • Embeddings turn text into vectors
  • Vector DB finds the relevant chunks
  • Chunks are pasted into the LLM prompt as context
  • LLM answers with citations

🔌Connectivity layer — MCP

  • One protocol, one wire format (JSON-RPC 2.0)
  • Any client (Claude Desktop, Cursor…) talks to any server
  • Servers expose tools, resources, prompts
  • Auth, isolation, and discovery are part of the spec

🤖Reasoning layer — Agentic RAG

  • Plans the work, doesn't just answer one query
  • Decides which tools / which retrievals to call
  • Reflects on results and self-corrects
  • Maintains short and long-term memory across steps

🎯Put together

  • Agentic RAG uses RAG to fetch knowledge
  • And uses MCP to invoke external tools
  • So an agent can read a contract, query a DB, file a ticket
  • All in one autonomous loop — that's the modern AI stack

Glossary

Every term you'll meet across the three decks, in one place. Skim once, refer back as needed.

LLM large language model

Deep neural net trained on huge text corpora that predicts the next word. GPT-4, Claude, Llama, Gemini.

Token sub-word unit

The atomic chunk an LLM sees — roughly ¾ of a word in English. Models have token limits.

Context window

The maximum number of tokens an LLM can read in one go. Bigger = more grounding fits in the prompt.

Embedding

A numeric vector representing the meaning of text. Similar meanings → vectors close in space.

Vector database

A store optimised for fast nearest-neighbour search over embedding vectors. e.g. OpenSearch k-NN, Pinecone, Weaviate.

Chunking

Splitting long documents into smaller passages before embedding. Bad chunking ruins retrieval.

BM25

Classic keyword-based ranking algorithm. Often combined with vector search → hybrid.

RRF reciprocal rank fusion

Standard way to merge BM25 and vector result lists into one ranked list.

Re-ranker

Second-stage model (often a cross-encoder) that scores query↔doc pairs more accurately than the first-stage retriever.

Cross-encoder

Reads the query and the document together in one forward pass. Slow but very accurate.

HyDE hypothetical doc embeddings

LLM writes a fake answer, you embed and search with that. Helps with short / vague queries.

MCP host

The AI app the user opens — Claude Desktop, Cursor, an IDE. Spawns one or more clients.

MCP client

A 1-to-1 connection from the host to a single server, isolated from other clients.

MCP server

A process that exposes one capability over the protocol (filesystem, Git, Slack, your DB…).

Tool / Resource / Prompt

The three primitives an MCP server can expose: callable functions, readable data, reusable prompt templates.

JSON-RPC 2.0

The simple request/response wire format MCP rides on. Requests, responses, and notifications.

Agent

An LLM wrapped in a loop that can decide what to do next, call tools, and observe results.

Reflection

Pattern where the agent critiques its own output and tries again. Boosts quality at the cost of latency.

Planning

Pattern where the agent first writes a step-by-step plan, then executes it.

Tool use

The agent calls external functions (search, calculator, API, code-runner) to act in the world.

Multi-agent

Several specialised agents collaborate — researcher, writer, critic, etc. — to solve one big task.

Memory (working / episodic / semantic / procedural)

Different layers an agent uses to remember context, past runs, learned facts, and how-to skills.

ADW agentic document workflow

End-to-end automation of a document-driven business process (e.g. invoice approval) by a team of agents.

Hallucination

When an LLM confidently states something that isn't true. Grounding (RAG) + verification reduce it.

Frequently asked

The questions most beginners ask in the first hour.

Do I need to know Python or maths to read these decks?

No. The decks include code snippets for illustration, but the explanations and visuals are built so a non-coder can follow them. You only need code if you want to implement what you see — and even then the snippets are short and well-commented.

What's the difference between RAG and MCP?

RAG is a technique: retrieve relevant text, then ground the LLM with it. MCP is a protocol: a standardised way for any LLM app to talk to any external data source or tool — including a RAG service. Many production systems use both: MCP as the wire, RAG as one of the tools the wire connects to.

Is Agentic RAG the same as "AI agents"?

Mostly yes. "AI agent" is the umbrella term; Agentic RAG is specifically the pattern that combines an agent loop with retrieval-augmented generation, so the agent can both find facts and act on them.

I'm short on time — which deck should I read first?

If you've only got one hour, read RAG → MCP, and skip Agentic RAG for later. RAG gives you the mental model for grounding, MCP gives you the mental model for connecting. Agentic RAG is exciting but only fully clicks once those two are in place.

Are these decks vendor-neutral?

Yes — they cover open standards (MCP from Anthropic, but governed openly; W3C WebMCP) and reference multiple model providers (GPT, Claude, Llama, Gemini, DeepSeek), vector DBs (OpenSearch, Pinecone, FAISS, ChromaDB), and frameworks (LangChain, LlamaIndex, FastMCP).

What if I want to build, not just read?

Each deck has worked examples and code snippets in Python / JS that you can copy. The RAG deck shows a full pipeline end-to-end; the MCP deck demonstrates Claude Desktop + Filesystem MCP; the Agentic RAG deck walks through an invoice-payment ADW.

How current is this material?

All three decks reflect the state of the art as of 2026, including the MCP 2026-07-28 RC, WebMCP, modern re-rankers (Cohere v3.5, FlashRank, ColBERT v2), and Agentic Document Workflows.
